# Pre-work - *Memory Game*

**Memory Game** is a Light & Sound Memory game to apply for CodePath's SITE Program. 

Submitted by: **AYOMIDE ISINKAYE**


## Required Functionality

The following **required** functionality is complete:

* [✅] Game interface has a heading (h1 tag), a line of body text (p tag), and four buttons that match the demo app
* [✅] "Start" button toggles between "Start" and "Stop" when clicked.
* [✅] Game buttons each light up and play a sound when clicked.
* [✅] Computer plays back sequence of clues including sound and visual cue for each button
* [✅] Play progresses to the next turn (the user gets the next step in the pattern) after a correct guess.
* [✅] User wins the game after guessing a complete pattern.
* [✅] User loses the game after an incorrect guess.

2. What was a challenge you encountered in creating this submission (be specific)? How did you overcome it? (recommended 200 - 400 words) 

    One of the biggest challenges I ran into was recording the project demonstration. I spent four days testing different software, thinking it would be a quick and simple step, except it wasn’t. I started with Loom, but every time I checked the recording, the game audio was either cut off or sounded glitchy. After multiple failed attempts, I started wondering if the problem was with my project itself. It was frustrating because the game was working fine, but I just couldn’t get a clean recording to showcase it properly.

    After trying different options, I switched to Zoom, which was a little better, but the audio still wasn’t great. Eventually, I decided to use the built-in screen recording feature on my MacBook, and that gave me the best results. It wasn’t perfect, but it was clear enough to submit. This whole experience taught me that sometimes, technical issues have nothing to do with the actual project, it’s just about finding the right tool for the job.

    Aside from the recording struggles, I also found myself second-guessing whether my project was "good enough" to turn in. The game itself wasn’t too complex, but I kept going back to tweak little details, making sure I wasn’t just rushing through it. I didn’t want it to feel incomplete or like I had done the bare minimum. I spent extra time refining the user interface and double-checking that the logic worked exactly as expected.

    In the end, I had to remind myself that I had already put in the effort and that my work was solid. I realized that sometimes, you just have to trust what you’ve done and know when to stop overthinking. This project definitely reinforced the importance of patience and adaptability; whether it’s dealing with technical difficulties or learning to have confidence in my own work.

    After completing my submission for the Light and Sound game, I have been thinking about several aspects of web development that I want to explore further to gain more knowledge. This is an area I am very eager to look into more deeply to provide users with the best experience possible.  

    Finally, I am currently exploring how to integrate external APIs. While the light and sound game was designed to be self-sufficient, I am now curious about how developers incorporate external data or services to enhance their applications. I am learning to use APIs effectively and securely, while also making sure they do not negatively affect performance or the overall user experience. These are my questions, which have motivated me to adopt a more proactive stance in my learning and skill development. 

4. If you had a few more hours to work on this project, what would you spend them doing (for example: refactoring certain functions, adding additional features, etc). Be specific. (recommended 100 - 300 words)
   
    If I had a few more hours to work on this project, I would concentrate on enhancing the gameplay to make it more dynamic and engaging through difficulty progression and musical elements. One of the first changes I would make is to implement a more sophisticated difficulty scaling system. As players advance, I would not only increase the sequence length but also introduce two or three more buttons. This should add variety and increase the challenge as players sharpen their memory and reaction skills.  

    Also, given my interest in music theory, I would also like to incorporate a more structured approach to the frequency mapping of the tones. Instead of assigning random tones to each button, I would consider using simple, recognizable melodies and their corresponding frequency values. As players progress, the patterns would start to resemble familiar tunes, making the experience more rewarding and interactive. This would add an extra layer of engagement, allowing players to recognize and anticipate patterns based on the musical structure rather than relying solely on memorization.  

    Furthermore, I would enhance the user interface by adding smoother transitions, subtle animations for button presses, and possibly visual cues that align with the musical progression. This would create a more immersive experience, blending memory training with interactive music composition. These improvements would make the game both more challenging and enjoyable, motivating players to push their limits while enjoying the musical elements.
